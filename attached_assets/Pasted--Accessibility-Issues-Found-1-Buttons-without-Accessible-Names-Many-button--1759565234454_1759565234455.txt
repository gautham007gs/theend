### 2. **Touch Targets Too Small**

* Some buttons (like round icon buttons with `p-1.5`) are **smaller than the recommended 48x48px target area**.
* This makes them hard to tap for users with mobility issues or on small devices.

---

### 3. **Manual Checks Needed** (not automatically failing, but Lighthouse suggests reviewing):

* Tab order follows logical order
* Visual order matches DOM order
* Focus not trapped in modal or region
* New content (like popups/chat) directs focus properly
* Proper ARIA roles for custom controls

# üöÄ Fix Plan

### A) **Fixing Buttons without Accessible Names**

1. Add **text** inside buttons (if possible).

   ```html
   <button class="bg-blue-500">Send</button>
   ```

2. If the button is **icon-only** ‚Üí use `aria-label`.

   ```html
   <button aria-label="Send message" class="hover:bg-green-400 rounded-full p-1.5">
     <svg><!-- icon --></svg>
   </button>
   ```

3. If multiple buttons exist with same icon ‚Üí give **unique labels**:

   ```html
   <button aria-label="Open settings"></button>
   <button aria-label="Start new chat"></button>
   ```

---

### B) **Fix Touch Target Sizes**

* Make touch areas at least **48x48px** (WCAG guideline).
* If the button icon is small, add `p-3` or `w-12 h-12 flex items-center justify-center`.
* Example:

  ```html
  <button aria-label="Close chat"
    class="hover:bg-green-400 rounded-full w-12 h-12 flex items-center justify-center">
    <svg><!-- close icon --></svg>
  </button>
  ```

---

### C) **Keyboard Navigation**

* Ensure **all buttons, links, inputs** are focusable (`<button>`, `<a href>` are fine by default).
* For custom controls (like divs with click handlers), **use role + tabindex**:

  ```html
  <div role="button" tabindex="0" aria-label="Expand menu"></div>
  ```

---

### D) **Focus Management**

* When opening a chat modal, **send focus** to the first interactive element:

  ```js
  document.getElementById("chatInput").focus();
  ```
* When closing, return focus back to the element that triggered it.

---

### E) **Landmarks & Semantic HTML**

* Wrap main sections in semantic tags:

  ```html
  <header>...</header>
  <main>...</main>
  <nav>...</nav>
  <footer>...</footer>
  ```

* Use ARIA landmarks if multiple roles:

  ```html
  <nav aria-label="Main navigation"></nav>
  ```
